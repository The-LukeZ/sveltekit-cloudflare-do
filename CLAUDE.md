## Project Overview

This is an npm package that bridges SvelteKit and Cloudflare Durable Objects. It provides both a Vite plugin and CLI tool to automatically export Durable Object classes to the Cloudflare Worker bundle generated by `@sveltejs/adapter-cloudflare`.

The package solves the problem where Durable Object classes need to be exported from `.svelte-kit/cloudflare/_worker.js` (a generated file), which is difficult to modify manually. Both the plugin and CLI are idempotent - they use a marker comment to prevent duplicate exports.

## Build and Development Commands

## Architecture

### Core Components

**Three-module architecture:**

1. **`src/core.ts`** - Core export logic
   - `exportDurableObjects()`: Main function that reads the generated worker file, checks for existing exports (via marker), validates Durable Object files exist, calculates relative paths, and appends export statements
   - Uses marker comment `// DURABLE_OBJECTS_EXPORT - do not remove` for idempotency
   - Handles both single and multiple Durable Object files
   - Path resolution: converts absolute DO file paths to relative paths from the worker directory

2. **`src/plugin.ts`** - Vite plugin wrapper
   - Default export: `cloudflareDoExporter(options)` function
   - Hooks into Vite's `closeBundle` lifecycle (runs after bundle is written)
   - Stores project root from `configResolved` hook
   - Throws error if export fails (unless already exported)

3. **`src/cli.ts`** - CLI tool
   - Executable via `sveltekit-cloudflare-do` command
   - Argument parsing: supports `--help`, `--version`, `--verbose`, `--worker`, `--do` (repeatable)
   - Config file loading: checks `.do-exporter.json` first, then `package.json` under `sveltekit-cloudflare-do` key
   - Priority: CLI args > file config > defaults

### Default Configuration

- Worker path: `.svelte-kit/cloudflare/_worker.js`
- Durable Objects path: `['src/lib/durable-objects.ts']`
- Marker: `// DURABLE_OBJECTS_EXPORT - do not remove`

### Package Structure

- Built as ESM module (`"type": "module"`)
- Main export: Vite plugin (default export)
- Secondary export: CLI utilities at `./cli`
- Bin command: `sveltekit-cloudflare-do` (points to `bin/cli.js`)
- Vite is a peer dependency (optional) - only required when using the plugin

### TypeScript Configuration

- Target: ES2020
- Module: ESNext with bundler resolution
- Outputs: dist/ with declarations and declaration maps
- Strict mode enabled

## Key Implementation Details

**Idempotency mechanism:** The tool checks for the marker comment before appending exports. If found, it skips the export operation.

**Path handling:** When generating export statements, the tool:
1. Resolves absolute paths for both worker and DO files
2. Calculates relative path from worker directory to DO file
3. Normalizes path separators for cross-platform compatibility (replaces `\` with `/`)
4. Uses these relative paths in the export statements

**Vite plugin lifecycle:** The plugin uses `closeBundle` hook rather than `writeBundle` to ensure all files are fully written before modifying the worker file.
